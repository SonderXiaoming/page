I've finally sat down to write my first post.

To be honest, I've been _planning_ this personal site for a long time — links, projects, things I wanted to showcase — but it always stayed at "I'll do it when I have time." That changed when I came across [Mizuki](https://github.com/LyraVoid/Mizuki), an **Astro**-based site framework that made the whole thing feel actually doable.

## Why this kind of framework

What I wanted wasn't just a blog template that works out of the box. I wanted:

- A solid theme and page structure so I don't have to design everything from scratch
- Enough flexibility to customize layouts, components, and styling
- The freedom to edit the code directly when needed, instead of being locked into a platform

Mizuki sits right in that sweet spot. Anime tracking, diary, albums, projects, skills, timeline — many of the pages a personal site needs are already there. At the same time, it's still your own codebase. Want a new feature or a small tweak? Open the editor and change it. For me, that beats both bare-bones templates and heavy CMS setups.

So the site I'd been putting off for ages finally has a version you can actually visit.

## What's on the site for now

For now, I've added some **basics** to get things standing:

- **About** — a short intro
- **Projects / Skills / Timeline** — filled in with what I could recall from GitHub and my background

Projects and tech stacks are hard to list completely in one sitting. A lot of entries will be **revised later**. A rough draft still beats a blank page.

## What's coming next

I plan to add more over time:

- **Anime** — watchlist and tracking
- **Diary** — casual notes and moments
- **Albums** — photos and galleries

Other pages and content will follow as needed. No rush to get everything perfect on day one.

## Closing thoughts

This site will probably change many times — content, layout, maybe a few features only I find fun. But at least it's moved from "I want to build this" to "it's live."

If you've been procrastinating on a personal site too, maybe try this: **ship a 60% version first, then polish it toward 90%.**

I'll keep writing here as things evolve.
